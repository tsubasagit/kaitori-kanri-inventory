## 1枚目: STEP 1 — 全工程をカバーする

```
ビジネスフローチャートのインフォグラフィック。白背景、フラットデザイン、日本語テキスト。
タイトル「STEP 1 — 全工程を1つのアプリでカバー」

中央に縦1列のフローチャート。すべてのステップが緑（#22c55e）の統一枠で囲まれ、1つのシステムであることを強調。左側に細い緑の縦ラインが全ステップを貫通する。

【フロー（上から下）】

① 「eKYC 本人確認」— アイコン: スマートフォンの画面に顔認証と免許証の画像が並んでいる。吹き出し「対面不要。スマホで顔写真+身分証を撮影 → AIが自動照合」。小さなバッジ「古物営業法対応」

② 「買取査定」— アイコン: タブレットに着物の写真と査定フォーム。吹き出し「コンディション5段階評価（S/A/B/C/D）、採寸（身丈・裄丈）、素材判定を1画面で入力」

③ 「QRコード/バーコード自動生成」— アイコン: QRコードが印刷されたタグ。吹き出し「HKM-YYYYMMDD-XXXX 形式で自動採番。ブラウザからラベル印刷」

④ 「在庫管理」— アイコン: ダッシュボード画面にステータス別の在庫数。吹き出し「in_stock / sold / returned / disposed をリアルタイム追跡」

⑤ 「販売（POS）」— アイコン: レジ画面でQRコードをスキャンしている。吹き出し「QRスキャン → カート → 決済。3タップで販売完了」

⑥ 「取引履歴・レシート」— アイコン: レシートのイラスト。吹き出し「S-YYYYMMDD-XXXX / P-YYYYMMDD-XXXX 形式で自動採番」

⑦ 「顧客管理」— アイコン: 顧客プロフィール画面。吹き出し「買取回数・購入回数・累計金額を自動集計」

【左端】緑の太い縦ライン。上端に「入口: eKYC」、下端に「出口: アフターフォロー」。ラインに沿って「ブラックボックス = ゼロ」のテキスト。

【右下の注釈】
「従来: 七福神（商品化後のみ）+ SATO（タグ印刷のみ）+ Notion（DBのみ）→ 3システムでも隙間あり」を薄いグレーの取り消し線で表示。
その下に緑文字「このアプリ: 全工程を1つでカバー。隙間ゼロ。」

色使い: 全ステップ枠 #22c55e（緑）、背景 白、アイコン #538bb0（青）、テキスト #1e293b
スタイル: Clean flat infographic, single column flow, Japanese text, professional, minimal
```

## 2枚目: STEP 2 — 1つのデータベースで統合管理

```
データベースアーキテクチャのインフォグラフィック。白背景、フラットデザイン、日本語テキスト。
タイトル「STEP 2 — 1つのデータベースで統合管理」

【中央: Firestoreアイコン（大きなDBシリンダー）】
オレンジ色（#f97316）のFirestoreロゴを大きく中央に配置。その周りに「Firestore」のテキスト。

【Firestoreを中心に、放射状に5つのコレクションが配置】

各コレクションは緑枠（#22c55e）のカード。カードの中にはテーブルのミニチュアイラスト（行と列が見えるDB風のアイコン）。

上: 「items（在庫）」— 小アイコン: 着物ハンガー。フィールド例: name, brand, category, condition, purchasePrice, sellingPrice, status, barcode
右上: 「customers（顧客）」— 小アイコン: 人物。フィールド例: lastName, firstName, phone, identityVerified, totalSellCount, totalBuyCount
右下: 「transactions（取引）」— 小アイコン: レシート。フィールド例: type(purchase/sale), items[], total, paymentMethod, receiptNumber
左下: 「counters（採番）」— 小アイコン: カウンター。フィールド例: receipts/YYYY-MM-DD/count
左上: 「users（スタッフ）」— 小アイコン: 名札。フィールド例: email, displayName, role

各カードからFirestoreアイコンへ緑の線で接続。「リレーション」を示す。

【下部: コスト比較テーブル】

横長の比較テーブル:

| 項目 | 七福神 + SATO | Notion | このアプリ |
| 初期費用 | パッケージ購入費 | 無料 | 無料 |
| 月額 | 保守費用 | ¥1,650〜/人 × 20名 = ¥33,000 | ¥0（Firebase無料枠） |
| カスタマイズ | 不可 | 限定的 | 自由自在 |
| データ所有 | ベンダー依存 | Notion社 | 自社所有 |

「このアプリ」列だけ緑ハイライト。他はグレー。

【右下の注釈】
「Firebase無料枠: 1GB ストレージ / 50K reads/day / 20K writes/day — 20名の店舗運用には十分」

色使い: Firestore #f97316（オレンジ）、コレクション枠 #22c55e（緑）、比較テーブルのハイライト #f0fdf4、テキスト #1e293b
スタイル: Clean flat infographic, database architecture diagram, Japanese text, professional, minimal
```

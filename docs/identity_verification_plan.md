# 古着在庫管理 — 本人確認方針書

## 背景

古物営業法（第15条第1項第3号 / 施行規則第15条第3項）により、古物商は買取時に相手方の本人確認が義務付けられている。違反時は**6月以下の懲役または30万円以下の罰金**。

取引形態によって求められる確認方法が異なる。

## 取引形態ごとの本人確認

### 対面取引（店頭買取）— 現在の運用

| 項目 | 内容 |
|---|---|
| 必要な対応 | 身分証（免許証等）の目視確認 + 古物台帳への記録 |
| 記録項目 | 氏名・住所・職業・年齢 |
| 保存期間 | 3年間 |
| eKYC | **不要** |

現在のシステムで対応済み（顧客登録時に本人確認情報を記録）。

### 非対面取引（宅配買取）— 今後の対応

警視庁が定める非対面取引の本人確認方法は全15種。  
参照: https://www.keishicho.metro.tokyo.lg.jp/tetsuzuki/kobutsu/kaisetsu/hitaimen.html

## 段階的導入プラン（推奨）

### Phase 1: 方法9（身分証コピー + 転送不要簡易書留 + 口座振込）

**最もシンプルで低コスト。宅配買取の開始時に推奨。**

```
[客] 身分証コピー + 商品を発送
         ↓
[店] 身分証コピーを受領・記録
         ↓
[店] 記載住所に「転送不要」簡易書留を送付
         ↓
[客] 簡易書留を受取（＝住所の実在確認）
         ↓
[店] 到達確認後、本人名義の口座に買取代金を振込
```

**必要なシステム対応:**
- 宅配買取フローの追加
- 身分証画像のアップロード・保存機能
- 簡易書留の送付ステータス管理（未送付 / 送付済 / 到達確認済）
- 振込先口座情報の登録
- 身分証コピーの保存（古物営業法で3年間保存義務）

**コスト:** 簡易書留の郵送代（1件あたり約400〜500円）のみ

---

### Phase 2: 方法14（eKYC — 顔写真 + 身分証画像）

**宅配買取の件数が増加し、郵送の手間・時間がボトルネックになった段階で導入。**

```
[客] スマホで顔写真 + 身分証（表・裏・厚み）を撮影・送信
         ↓
[eKYCサービス] AIが本人マッチング + 真贋判定
         ↓
[店] 確認完了 → 本人名義の口座に買取代金を振込
```

**メリット:**
- 郵送不要、即時完了
- なりすまし防止（AIによる顔マッチング）

**eKYCサービス候補:**

| サービス | 提供元 | 導入方式 | 料金 | API公開 |
|---|---|---|---|---|
| GMO顔認証eKYC | GMOグローバルサイン | REST API連携 | 要問い合わせ | **非公開**（契約後に仕様書提供） |
| ProTech ID Checker | ショーケース | タグ設置（ノーコード） | 月額18,000円〜 + 従量 | **非公開** |

**課題:**
- いずれのサービスもAPIドキュメントは一般公開されていない
- テスト・検証を行うには、サービス提供元に問い合わせて以下を取得する必要がある:
  - **API接続URL（エンドポイント）**
  - **テスト用ID / パスワード（APIキー）**
  - **API仕様書**
- 契約前のトライアル/サンドボックス環境の有無も要確認

---

### 共通: 方法10（リピーター対応 — ID/パスワード認証）

**Phase 1・2いずれでも適用可能。初回確認済みの顧客は2回目以降の本人確認を省略。**

```
[初回] 方法9または方法14で本人確認を実施
         ↓
[システム] 顧客に「確認済み」フラグ + ログインID/PWを付与
         ↓
[2回目以降] ID/PW入力のみで買取受付可能
```

**必要なシステム対応:**
- 顧客アカウント機能（ID/PW発行・ログイン）
- 「初回本人確認済み」フラグの管理
- 顧客向けマイページ（任意）

## まとめ

| フェーズ | 方法 | 条件 | コスト | 所要時間 |
|---|---|---|---|---|
| Phase 1 | 方法9: 身分証コピー + 転送不要簡易書留 + 口座振込 | 宅配買取の開始時 | 郵送代のみ | 数日〜1週間 |
| Phase 2 | 方法14: eKYC（顔写真 + 身分証画像） | 件数増加時 | 月額費用 + 従量課金 | 即時完了 |
| 共通 | 方法10: ID/PW（リピーター） | 初回確認済みの顧客 | なし | 即時 |

**次のアクション:**
1. Phase 1の機能をシステムに実装
2. eKYCサービスに問い合わせ、テスト用のAPI接続URL・ID・パスワード・仕様書を入手
3. トライアル環境で検証後、Phase 2の導入判断
